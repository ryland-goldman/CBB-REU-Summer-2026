## 3.6 GDF2A

```
gdf2a [-hHv] [-w width] [-o outfile] infile [arrays...]
```
Writes a GDF-file in ASCII form to the terminal or to a file.

```
-h
```
Give short help and quit.

```
-H
```
Give long help and quit.

```
-v
```
Verbose mode. Instructs the program to write additional information, such as data type.

```
-w width
```
Specify the width (in characters) of each field. Default 10.

```
-o outfile
```
Write output to a file named `outfile`. If not specified, output is written to `stdout`.

```
infile
```
Filename of the GDF file to convert to ASCII. If not specified, the program reads the GDF file from `stdin` unless `arrays` are listed. In the latter case, a dash `-` must be specified to read from from `stdin`.

```
arrays
```
The names of the arrays to be written to `outfile`. If not specified, all arrays are written.

Note: To reduce the size of the ouputfile, the default number of digits written is set to single precision. The `–w 16` option can be used to instruct GDF2A to write output in double precision.

Because GDF2A can read from `stdin` and write to `stdout`, it can be piped after any command that writes a GDF file to `stdout`.
