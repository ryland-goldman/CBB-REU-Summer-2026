## 3.5 Rotations

Reference Coordinates view – Z vs. X

In G4beamline, all rotations are specified relative to the fixed coordinate axes of the parent volume into
which a given element is being placed. This is the natural way to think of placing an object into a room,
where the object is the element being placed, and the room is the parent volume into which it is placed.

Rotations can be specified in the place command, and in a few other commands. They are specified by a
rotation parameter, the value of which is a comma-separated list of rotations around the parent’s
coordinate axes, which are applied in order (left-to-right) to the object; values are in degrees. So
“rotation=Y90,X30” means:
1. Start with the object’s local coordinate axes aligned with the parent’s coordinate axes
2. Rotate the object (and its local coordinate axes) by 90 degrees around the parent’s Y axis
3. Then rotate the object (and its local coordinate axes) by 30 degrees around the parent’s X axis.
Negative and decimal values of the angle are permitted. The direction of a positive rotation is given by
the right-hand rule: use your right hand to grasp the rotation axis with your thumb in its positive
direction; a positive rotation goes in the direction of your fingers.
