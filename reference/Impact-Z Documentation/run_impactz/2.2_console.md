## 2.2. Console

When the code is running, the console shows the order of elements, element type, position Z, element steps, and total steps.

### Console Output Fields

Each line of console output during simulation contains:

| Field | Description |
|-------|-------------|
| Element order | The sequential index of the current element being processed |
| Element type | The numeric element type code (e.g. 0 = drift, 1 = quad, 104 = superconducting cavity) |
| Position Z | The z position (in meters) at the current step |
| Element steps | The current step number within the element |
| Total steps | The cumulative total number of steps taken |

### Example Output

```
pass generating initial distribution...
pass setting up lattice...
enter element (type code):         1         0
zedge:    0.000000000000000
j, nstep, z       1    1    6.208229839349999E-002
enter element (type code):         2         1
j, nstep, z       1    2    7.874896506016667E-002
j, nstep, z       2    3    9.541563172683333E-002
j, nstep, z       3    4    0.112082298394000
enter element (type code):         3         0
zedge:    0.112082298394000
enter element (type code):         4       104
```

- **"The order of the element"**: sequential counter of elements processed (1, 2, 3, ...)
- **"Element type"**: the type code from the lattice section (0, 1, 104, etc.)
- **"Position Z"**: the z coordinate in meters at the leading edge (`zedge`) or at each step
- **"Element steps"**: step counter within the current element (`j`)
- **"Total steps"**: cumulative step counter across all elements (`nstep`)
