## 1.3. particle.in

The `particle.in` file contains initial particle distribution data. The first line gives the total particle number.

Each subsequent line contains the 6D phase space coordinates and particle properties for one macroparticle, in the following column format:

| Column | Description |
|--------|-------------|
| 1 | x (normalized by c/omega) |
| 2 | Px (normalized by mc) |
| 3 | y (normalized by c/omega) |
| 4 | Py (normalized by mc) |
| 5 | phase (radian) |
| 6 | kinetic energy deviation (E0 - E) normalized by mc² |
| 7 | q/m |
| 8 | charge per macro-particle (Coulomb per particle) |
| 9 | particle id |

### Example particle.in File

```
1 00000 0.0 0.0
1  0.3638365E-02  0.1930983E-03  -0.2149497E-01   0.3292235E-03   0.5304560E-01   0.2591784E-04   0.1065789E-08   0.6153846E-15   0.2900000E+02
2 -0.1299163E-02  0.6297654E-04  -0.2443249E-01  -0.2443178E-03   0.6781523E-01  -0.3582175E-03   0.1065789E-08   0.6153846E-15   0.6800000E+02
3 -0.4718904E-01  0.1736739E-03  -0.2149297E-01   0.6414229E-03   0.4358065E-01  -0.4211717E-04   0.1065789E-08   0.6153846E-15   0.7600000E+02
...
```

> **Note on normalization constants:** Particle data written with the `"-2"` marker element are dimensionless in IMPACT internal units. The columns are:
> - x normalized by c/omega
> - Px normalized by mc (must be divided by gamma\*beta to convert to unit radian)
> - y normalized by c/omega
> - Py normalized by mc (must be divided by gamma\*beta to convert to unit radian)
> - phase (radian)
> - kinetic energy deviation normalized by mc²
> - q/m (for an electron, q = -1, m = 0.511001e6)
> - charge per macro-particle (Coulomb per particle)
> - particle id
